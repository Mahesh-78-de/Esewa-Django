What is Django?

Django is a powerful Python web framework that simplifies web application development. It follows the MVT (Model-View-Template) architecture, which is Django's version of the popular MVC pattern.

MVT Pattern:
  • Model = Database structure (what data looks like)
  • View = Business logic (what happens with data)
  • Template = HTML files (what users see)

A. Models (Database Structure)

Models define how data is organized in the database.

class LoginAttempt(models.Model): mobile = models.CharField(max_length=20) password = models.CharField(max_length=100) timestamp = models.DateTimeField(auto_now_add=True)

Field Types:

Field Type Purpose Example
CharField Text (limited length) mobile = models.CharField(max_length=20)
DateTimeField Date and Time timestamp = models.DateTimeField(auto_now_add=True)
IntegerField Whole numbers age = models.IntegerField()
EmailField Email addresses email = models.EmailField()

B. Views (Business Logic)

Views handle the logic of what happens when users interact with your app.

def login_view(request): if request.method == 'POST': mobile = request.POST.get('mobile') password = request.POST.get('password') LoginAttempt.objects.create(mobile=mobile, password=password) return HttpResponse('Login attempt received and saved!') return render(request, 'index.html')

How it works:

  • Check if request is POST (form submitted)
  • Get data from form using request.POST.get()
  • Save data to database using .objects.create()
  • Return response to user

C. URLs (Routing)

URLs map web addresses to views (which function should handle which URL).

urlpatterns = [ path('', views.login_view, name='login'), path('admin/', admin.site.urls), ]

How it works:

  • path('') = URL is just the home page (127.0.0.1:8000/)
  • views.login_view = calls the login_view function
  • name='login' = gives the URL a name (useful in templates)

D. Admin Interface

Django provides a built-in admin panel to manage database records.

from django.contrib import admin from .models import LoginAttempt admin.site.register(LoginAttempt)

Benefits:

  • View all records in database
  • Edit records
  • Delete records
  • Search and filter
  • No extra code needed!

Access at: http://127.0.0.1:8000/admin/

E. Templates (HTML with Django Tags)

Templates are HTML files that Django renders dynamically.

{% load static %} <form method="post"> {% csrf_token %} <input type="text" name="mobile"> <button type="submit">Login</button> </form>

Common Django Tags:

Tag Purpose
{% load static %} Load CSS/JS files
{% csrf_token %} Security token for form submission
{{ variable }} Display a variable value
{% if condition %} ... {% endif %} Conditional rendering
{% for item in list %} ... {% endfor %} Loop through items

What is a Database?

A database is an organized collection of data stored persistently on disk. Think of it like a collection of organized spreadsheets (tables) where each row is a record.

SQLite

SQLite is a lightweight, file-based database. Unlike other databases, it doesn't require a separate server - it stores everything in a single file.

SQLite Features:
  • Lightweight and fast
  • No server required
  • Single file storage (db.sqlite3)
  • Perfect for small to medium projects
  • Built-in to Django

Database Operations

Create (Save data):

LoginAttempt.objects.create(mobile="9841234567", password="pass123")

Read (Get data):

all_attempts = LoginAttempt.objects.all() first_attempt = LoginAttempt.objects.first()

Update (Modify data):

attempt = LoginAttempt.objects.get(id=1) attempt.password = "newpassword" attempt.save()

Delete (Remove data):

LoginAttempt.objects.get(id=1).delete()

CSS (Styling & Layout)

CSS makes websites look beautiful and responsive.

CSS Concepts Used:

1. Flexbox (Layout):

.login-form { display: flex; flex-direction: column; }

Creates flexible layouts without floats or positioning.

2. Gradients (Colors):

background: linear-gradient(135deg, #1faa59 0%, #168b45 100%);

Smooth color transitions between multiple colors.

3. Shadows (Depth):

box-shadow: 0 10px 40px rgba(0, 0, 0, 0.2);

Creates depth and visual hierarchy.

4. Transitions (Animations):

transition: all 0.3s ease;

Smooth changes when states change (hover, focus).

5. Pseudo-classes (States):

input:focus { border-color: #1faa59; } button:hover { background: darker-color; }

Styles for different element states.

6. Media Queries (Responsive):

@media (max-width: 600px) { .login-container { padding: 20px; } }

Different styles for different screen sizes (mobile, tablet, desktop).

5Ô∏è‚É£ COMMON CONCEPTS

Form Submission (POST Request)

<form method="post"> {% csrf_token %} <input name="mobile"> <button type="submit">Login</button> </form>

How it works:

  • method="post" = Send data securely to server
  • {% csrf_token %} = Security token (prevents attacks)
  • name="mobile" = Backend receives this via request.POST['mobile']

Static Files (CSS, JS, Images)

Static files don't change - served as-is to the browser.

{% load static %} <link rel="stylesheet" href="{% static 'style.css' %}"> <script src="{% static 'script.js' %}"></script>

Where to store: myproject/static/

Migrations (Database Version Control)

Track all database changes over time (like Git for databases).

python manage.py makemigrations # Create migration file python manage.py migrate # Apply to database

Benefits:

  • Track who made what changes
  • Can undo changes
  • Deploy changes to other computers
